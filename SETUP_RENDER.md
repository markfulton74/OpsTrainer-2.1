# Render Deployment Setup

## Environment Variables to Set in Render Dashboard

Go to: Render Dashboard → opstrainer service → Environment

### Required — set these manually:

| Variable | Value | Source |
|----------|-------|--------|
| `DEEPSEEK_API_KEY` | Your DeepSeek API key | https://platform.deepseek.com |
| `STRIPE_SECRET_KEY` | `sk_live_...` | Stripe Dashboard |
| `STRIPE_WEBHOOK_SECRET` | `whsec_...` | Stripe Webhooks |
| `EMAIL_HOST` | `mail.opstrainer.co.za` | Your hosting provider |
| `EMAIL_USER` | `noreply@opstrainer.co.za` | Your hosting provider |
| `EMAIL_PASSWORD` | Your email password | Your hosting provider |
| `APP_URL` | `https://opstrainer.onrender.com` | Your Render URL |

### Auto-generated by render.yaml (no action needed):

- `JWT_SECRET`
- `JWT_REFRESH_SECRET`

### Fixed values (already in render.yaml):

- `NODE_ENV=production`
- `PORT=3000`
- `DB_PATH=./data/opstrainer.db`
- `DEEPSEEK_API_URL=https://api.deepseek.com/v1/chat/completions`
- `DEEPSEEK_MODEL=deepseek-chat`
- `EMAIL_FROM=noreply@opstrainer.co.za`

## ⚠️ Note on SQLite on Render Free Tier

Render's free tier uses **ephemeral storage** — the SQLite database resets on every deploy.

**Options:**
1. Upgrade to Render Starter ($7/month) and add a persistent disk at `/data`
2. Or migrate to Render PostgreSQL (free 90-day trial, then $7/month)

For production use, persistent storage is essential.

## Uploading Your Logo & Certificates

Push files to the `assets/` folder in this repo:

```
assets/branding/logo.png          ← Your logo
assets/branding/favicon.ico       ← Browser favicon
assets/certificates/template-standard.pdf   ← Certificate template
```

They will be served at `https://opstrainer.onrender.com/assets/...`
